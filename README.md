# 五子棋人机对战

一个简洁优雅的网页版五子棋游戏，支持玩家与 AI 对战。

## 功能介绍

- **人机对战**: 玩家执黑子（先手），AI 执白子（后手）
- **智能 AI**: 基于位置评分算法的 AI 对手
- **悔棋功能**: 支持撤销上一步操作
- **重新开始**: 一键重置游戏
- **自动胜负判定**: 五子连珠自动检测并显示获胜方
- **视觉反馈**: 最后一步棋高亮显示，当前执子提示

## AI 算法说明

### 评分系统

AI 采用基于位置评分的启发式算法，主要包含以下核心组件：

1. **候选落子点生成**
   - 仅考虑已有棋子周围 2 格范围内的空位
   - 避免遍历整个棋盘，提高计算效率
   - 选取评分最高的 15 个候选点进行评估

2. **局面评估因素**
   - **连子数量**: 评估在四个方向（横、竖、斜、反对角）上的连续棋子数
   - **开放端数**: 评估棋子两端是否为空位（0-2 个开放端）
   - **攻防权重**: 优先考虑自己的进攻机会，同时防守对方的威胁

3. **棋型评分表**
   | 棋型 | 评分 |
   |------|------|
   | 五连 | 100000 |
   | 四连（双端开放） | 10000 |
   | 四连（单端开放） | 1000 |
   | 三连（双端开放） | 1000 |
   | 三连（单端开放） | 100 |
   | 二连（双端开放） | 100 |

4. **决策逻辑**
   - 优先完成自己的五连
   - 其次阻断对方的五连/四连
   - 最后选择评分最高的落子点

## 技术细节

- **前端框架**: 原生 HTML5 + CSS3 + JavaScript（无外部依赖）
- **棋盘规格**: 15×15 标准棋盘
- **渲染方式**: DOM 操作实时渲染
- **响应式设计**: 支持桌面和移动设备
- **代码结构**: 单文件实现，约 540 行代码

## 部署说明

### 本地运行

```bash
# 克隆项目
git clone https://github.com/williamxiewz-bot/gomoku.git
cd gomoku

# 启动本地服务器
python3 -m http.server 8080
```

访问 http://localhost:8080

### 静态部署

由于是纯静态文件，可直接部署到任何静态托管服务：

- GitHub Pages
- Vercel / Netlify
- Nginx / Apache

## 玩法说明

1. 点击棋盘上的交叉点落子
2. 黑方先行，玩家先手
3. AI 自动思考并落子（约 300ms 延迟）
4. 五子连珠即获胜
5. 可使用"悔棋"按钮撤销操作
6. 可使用"重新开始"按钮重置游戏
